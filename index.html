iOS 基础(1)- Property 修饰符

发表于 2017-08-15

1.atomic & nonatomic

多线程相关，指定属性的读取操作是否为原子操作，即是否线程安全。
atomic为默认选项，表示线程安全，至少在当前的存取器上是安全的，意味着同一时间只能有一个线程访问实例变量。线程安全涉及到加锁机制，会消耗大量资源损耗性能，所以效率会比较低。

nonatomic表示非线程安全，修饰的实例变量可以同时被多个线程访问。跟atomic相比，效率比较高，可以提高访问性能。适合内存比较小的移动设备。

2. readwrite & readonly

存取器控制，
readwrite：默认选项，表示属性同时拥有同时拥有setter方法和getter方法。
readonly：表示属性只拥有getter方法，没有setter方法。

3. strong & weak

在 ARC 下用于属性的内存管理，用来修饰对象，不能用来修饰基本的值数据类型，如int，float等。

strong：表示一种拥有关系，属性和所赋的值之间是强引用关系。赋值时，先保留新值，增加新值的引用计数，然后再释放旧值减少旧值的引用计数。strong用来修饰对象，不用修饰基本的值数据类型。

weak表示一种非拥有关系，属性和所赋的值之间是弱引用的关系。赋值时，不会增加新值的引用计数，也不减少旧值的引用计数。weak声明的属性在引用计数减少为 0 被释放后，会自动设置为 nil，避免野指针的错误。
weak常被用在delegate等容易出现循环引用的场景。

4. retain

在 MRC 下用于属性的内存管理，和strong的用法一致。

5. assign

assign表示单纯的复制（值复制），赋值时，不会更改新值和旧值的引用计数。用于值类型，如int,float,double和NSInteger，CGFloat等，还包括不存在所有权关系的对象，如常见的delegate。

assign也可以用来修饰对象，如NSString等类型对象，由于在赋值过程中没有改变新值以及旧值的引用计数，所以当新值的引用计数为 0 时，编译器不会将该属性置为nil，指针还会指向之前被销毁的内存，这时访问该属性会产生野指针错误并崩溃，因此使用assign修饰的类型一定要为值类型。

6. copy

copy修饰的对象，与strong相同，都是强引用关系，不同的是，赋值时copy修饰的属性不会保留新值，而是将新值在内存中拷贝了一份。

Git 的基本命令

发表于 2017-08-12

基本

分支

1. git branch                      //查看本地当前分支
2. git branch new-feature          //在本地创建名称为「new-feature」的分支
3. git checkout new-feature        //切换到名称为「new-feature」的本地分支
4. git branch -d new-feature       //删除名称为「new-feature」的本地分支
5. git branch -r                   //查看远程分支列表   
6. git branch -d -r new-feature    //删除名称为「new-feature」的远程分支

pull

1. git pull origin new-feature     //将名称为「new-feature」的远程分支的内容 pull 至本地

push

1. git add .                    //把我们要提交的文件信息添加到索引库中,点所有文件
2. git add filename             //把指定的文件信息添加到索引库中
3. git commit -m "comments"     //依据索引库中的内容来进行文件的提交, comments为提交的备注信息
4. git commit -a -m "comments"  //提交工程目录下的所有文件, comments为提交的备注信息
5. git push origin new-feature  //将内容 push 至名称为「new-feature」的远程分支

merge

1. git merge new-feature        //把名称为「new-feature」的分支内容合并到当前分支

一般情况下，使用 git 开发的步骤为:

1. 在自己分支开发 
2. git add & git commit 
3. git pull(这一步是从远程分支拉取和自己的本地分支合并) 
4. 如果有冲突，解决冲突 
5. git push

iOS 动画篇： UIView Animation

发表于 2016-09-26

UIVIew Animation 是 iOS 提供的最基础的一组用于实现 UIView 动画的类库。在 UIView Animation 中，可以改变的属性有：

  • frame
  • bounds
  • center
  • alpha
  • transform
  • …

UIView 类提供了大量的动画 API ，这些 API 集中在三个 Category 里，分别是：

  1. UIView (UIViewAnimation) - basic animation 基础动画
  2. UIView (UIViewAnimationWithBlocks) - basic animation 基础动画
  3. UIView (UIViewKeyframeAnimations) - keyframe animation 关键帧动画

1. UIViewAnimation

方法比较简单，使用时先 beginAnimations（传入的 animationID 作为该动画的标识，可以在 delegate 中清楚的识别到该动画）,然后设置动画的各项属性，如 duration, delegate 等，设置完成后 commitAnimations。

需要说明的是，UIViewAnimationCurve 表示动画的变化规律：

  • UIViewAnimationCurveEaseInOut: 开始和结束时较慢
  • UIViewAnimationCurveEase: 开始时较慢
  • UIViewAnimationCurveEaseOut: 结束时较慢
  • UIViewAnimationCurveLinear: 整个过程匀速进行

2. UIViewAnimationWithBlocks

UIViewAnimationWithBlocks 是在 iOS 4.0 时推出的基于 block 的 Animation Category，较 UIViewAnimation 来说，使用起来更加便捷。

这是 UIViewAnimationWithBlocks 中最常用的三种方法，使用 block 方式实现基本动画，可以设置 duration持续时间，delay延迟时间，UIViewAnimationOptions枚举项和completion动画结束的回调。

时间函数

动画的速度曲线是由时间函数（ timing function ）控制的。

弹簧动画

iOS 7.0 增加的方法，新添了两个参数，springDamping和initialSpringVelocity。

  • springDamping：弹性阻尼，取值范围时 0 到 1，越接近 0 ，动画的弹性效果就越明显；如果设置为 1，则动画不会有弹性效果。
  • initialSpringVelocity：视图在动画开始时的速度，>= 0。

    在 initialSpringVelocity 为 0 ，damping 分别为 0.4，0.6，0.8 的情况下效果如上图，可见阻尼越小，弹簧效果越明显。

    在 damping 为 1 ，initialSpringVelocity 分别为 0，5，30 的情况下效果如上图，可见初始速度越大弹簧效果越明显，弹动的幅度越大。

3. UIViewKeyframeAnimations

UIViewAnimationWithBlocks 推出于 iOS 7.0，用来实现帧动画 。基础动画只能将 UIView的属性从一个值变化到另一个值，而关键帧动画可以包含任意一个关键帧，使 UIView在多个值之间进行变化。关键帧动画可以看成是若干个连续执行的基础动画。

其中animateKeyframesWithDuration:delay:options:animations:completion:参数的使用方法与基础动画大致相同，只是基础动画的 options 是 UIViewAnimationOptions 类型，而关键帧动画的 options 是 UIViewKeyAnimationOptions 类型。另外，关键帧动画的持续时间（ duration ）是整个动画的持续时间，也就是所有关键帧持续时间的总和。
addKeyframeWithRelativeStartTime:relativeDuration:animations:中的第一个参数（ relativeStartTime ）是相对起始时间，表示该关键帧开始执行的时刻在整个动画持续时间中的百分比，取值范围是［0-1］。第二个参数（ relativeDuration ）是相对持续时间，表示该关键帧占整个动画持续时间的百分比，取值范围也是［0-1］。
