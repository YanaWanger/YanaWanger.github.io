Git 的基本命令

发表于 2017-08-12

基本

分支

1. git branch                      //查看本地当前分支
2. git branch new-feature          //在本地创建名称为「new-feature」的分支
3. git checkout new-feature        //切换到名称为「new-feature」的本地分支
4. git branch -d new-feature       //删除名称为「new-feature」的本地分支
5. git branch -r                   //查看远程分支列表   
6. git branch -d -r new-feature    //删除名称为「new-feature」的远程分支

pull

1. git pull origin new-feature     //将名称为「new-feature」的远程分支的内容 pull 至本地

push

1. git add .                    //把我们要提交的文件信息添加到索引库中,点所有文件
2. git add filename             //把指定的文件信息添加到索引库中
3. git commit -m "comments"     //依据索引库中的内容来进行文件的提交, comments为提交的备注信息
4. git commit -a -m "comments"  //提交工程目录下的所有文件, comments为提交的备注信息
5. git push origin new-feature  //将内容 push 至名称为「new-feature」的远程分支

merge

1. git merge new-feature        //把名称为「new-feature」的分支内容合并到当前分支

一般情况下，使用 git 开发的步骤为:

1. 在自己分支开发 
2. git add & git commit 
3. git pull(这一步是从远程分支拉取和自己的本地分支合并) 
4. 如果有冲突，解决冲突 
5. git push
